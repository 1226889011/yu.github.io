    git与github

    git init 初始化一个文件夹为一个git仓库

    ls -a查看文件夹下文件

    设置签名：

    git config [–global] user.name 用户

    git config [–global] user.email 邮箱

    提交操作

    git add 文件名 （通知git将要提交这个文件，还没有提交上去git）

    git commit -m”日志” hello.java 把文件提交到本地仓库

    git reset HEAD 取消提交

    git status 查看git状态

    git rm –cached 文件名 将文件从缓存中删除

    git diff：查看当前版本发生的差异

    工作区（文件夹），暂存区，Git仓库

    基本的Git工作流程：

    1、在工作目录中修改文件

    2、暂存文件，将文件的快照放入暂存区域。git add hello.java

    3、提交更新，找到暂存区的文件，将快照永久性储存到Git仓库目录。Git commit -m “日志” 文件名

    1、用 git log可以查看提交历史，可以获取版本号

    2、使用git log -pretty=oneline 查看简介的历史版本

    3、用 git reset –hard commit id 可以退回到某个版本, commit id是版本号

    4、退回之后，相当于时光穿梭回过去，所以看不见未来的版本，git log 看不到信息。所以采用git reflog

    可以解决这个问题，该命令可以显示回退到历史版本需要几步

    git reset –hard 开头的id（再次选择版本）

    撤销操作：

    1、如果文件只是在工作区修改了，我们想要撤销这次修改可以用 git checkout – 文件名

    用 cat 文件名 查看文件内容

    2、如果文件已经在缓存区了，先用 git reset 文件名 或者 git reset head 文件名 重置一下

    然后用git checkout – 文件名 进行覆盖

    场景：本地修改，未add

    撤销：git checkout – 文件名 撤销本地修改

    场景： 本地修改，已经add 未commit

    git reset head hello.java

    git checkout – 文件名

    删除文件

    git rm 文件名

    git commit -m “操作日志” 提交本次删除

    想恢复

    git reflog

    git reset –hard 版本号

    分支开发

    git branch 分支名 新建分支

    git branch -v 查看分支信息

    git checkout 分支名

    把分支合并到master（主干）下

    git checkout master

    git merge 分支名

    关联远程仓库

    git remote -v 查看本地仓库关联远程仓库关联信息

    git remote add origin 远程仓库地址 关联远程仓库

    git push -u origin master 把master分支提交到远程仓库

    上传代码是需要权限的，下载代码是不需要权限的，只要是公开的都可以下载。

    第一个问题

    上传步骤：
    git add .
    git commit -m “提示消息”
    git push origin master
    出错：
    ! [rejected] master -> master (fetch first) error: failed to push some refs to ‘ 。。。’
    出现这个问题是因为github中的README.md文件不在本地代码目录中，可以通过如下命令进行代码合并
    git push –rebase origin master

    克隆项目(下载项目)

    git clone https://github.com/cyhbyw/springBoot_atguigu.git(克隆地址)

    把远程仓库的代码拉到本地

    git pull origin master （git pull 别名 分支名）
