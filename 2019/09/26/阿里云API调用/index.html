    阿里云API调用

    1、什么是API

    API（Application Programming Interface,应用程序编程接口）是一些预先定义的函数，目的是提供应用程序于开发人员基于某软件或硬件得以访问一组例程的能力，而又无需访问源码或理解内部工作机制的细节。

    2、API特点

    1、API是一个明确定义的接口，可以为其他软件提供特定的服务

    2、API可以小到只包含一个单独的函数，也可以大到包含数以百计的类、方法、全局函数、数据类型、枚举类型和常量等

    3、API的实现可以是私有的，也可以是开源的

    3、为什么要使用API

    1、快速扩展功能

    2、避免“造轮子”，提高开发效率

    3、降低模块之间的耦合度

    4、阿里云API市场提供1000+API服务

    5、Web API协议

    Web API一般采用HTTP作为底层协议，HTTP请求机制如下：（基于http协议的好处是客户端与服务端的开发语言可以无关，有利于微服务分布式系统的开发）

    1、客户端向服务器发送一个请求

    2、服务器给客户端一个响应，告诉客户端是否可以完成它的请求工作

    6、Http请求包含的内容

    1、URL（API调用地址）

    2、请求方式

    3、Headers（请求头）

    ​ 提供了请求的元信息，是一个简单的项目列表，其中有客户端发送请求的时间和请求主体大小、身份认证等信息。

    4、Body(请求主体)

    ​ 包含了客户端希望发送给服务器的数据

    7、API请求方式

    1、GET:请求服务器获取一个资源

    2、POST：请求服务器创建一个新的资源

    3、PUT：请求服务器编辑或更新一个已存在的资源

    4、DELETE:请求服务器删除一个资源

    8、状态码–状态成功

    当成功调用API后，除了返回数据之外，还会包含一个状态码，处理成功返回2xx。

    9、状态码–服务端错误码

    API未调用成功，则返回错误码。服务端错误码是5xx，表示服务不可用（一般重试或者联系商品页面的API服务商）

    10、状态码–客户端错误码

    客户端错误码为4xx，表示业务报错。此时一般为参数错误、签名错误、请求方式有误或被限流控制等业务类错误。建议详细查看错误码，针对性解决问题。

    11、JSON数据格式表示方法

    1、表示对象

    JSON最常用的格式是对象的键值对

    2、表示数组

    和普通的JS数组一样，JSON表示数组的方式是使用方括号[]

    12、API简单身份认证(APPCODE方式)

    13、API签名认证(AppKey&AppSecret)

    14、API调用步骤

    1、获取API文档

    2、创建应用

    3、获取授权

    4、调用API

    要调用API需要三个基础条件

    1、API：您即将要调用的API，明确API参数定义。

    2、应用app：作为您调用API时的身份，有AppKey和AppSecret用于验证您的身份。

    3、API和App的权限关系：App想调用某个API需要具有该API的权限，这个权限通过授权的功能来建立。

    15、API调用注意事项

    1、每个账号下APP个数上线为10个，App名称应为账号下唯一

    2、调用API的流控限制为，单个IP，QPS不超过100.

    3、你有权操作购买的API与APP的授权和解除授权。由服务提供方授权给你的APP的API，你无权操作解除权限

    4、你的请求需要包含签名信息

    API发送请求之后，正常就返回响应头和响应内容

    有返回头，返回code，返回结果体
